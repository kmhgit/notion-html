2.0 (參考 : Day2 - Node.js 基礎)

2.1 Node.js 基礎

.1 同前(1.3.1-.2)，打開終端機（Terminal）或者命令提示字元（CMD），

在 C:\Users\…>＿(輸入) node 〈enter〉， 出現以下畫面 (即進入 REPL 模式)

.2 > 這個模式叫做「Node REPL（Read-Eval-Print-Loop）」，簡單來講就是進入 Node.js 環境，我們可以直接在這個模式下輸入 JavaScript 的程式碼並按下 Enter 直接執行，每一次輸入完畢後 Node.js 都會幫我們執行並且印出結果，這樣的模式就是 REPL。

.3 在REPL模式下(畫面出現 “>_”， 而非”C:\Users\…>_”)，試著輸入以下程式碼並按下<enter>

> console.log('Hello World')

接著就會看到以下畫面

> console.log('Hello World')
Hello World
undefined
>

你可以想像成: 在REPL模式下，我們是在瀏覽器的 Console (控制臺)輸入程式碼，然後按下 Enter，瀏覽器就會幫我們執行並且印出結果。

.4 如果想要退出 REPL 模式的話，只需要在 >_ 按下<Ctrl+c>兩次，或者直接輸入 .exit ，就可以退出REPL模式，回到終端機 C:\Users\…>＿

.5 當然 node 這個指令不只有進入 REPL 的功能而已，其實它還有其他參數可以使用，像是最常見的 -v(version) 參數，這個參數可以讓我們查看目前 Node.js 的版本之外，額外介紹一個參數，也就是 --inspect(檢查)，這個參數可以讓我們在 Node.js 中使用 Chrome DevTools 來偵錯。

在終端機 C:\Users\…>＿輸入
node --inspect <enter>

接下來終端機會出現以下畫面

C:\Users\…> node --inspect
Debugger listening on ws://127.0.0.1:9229/508515c1-f25d-4445-90d6-8b28ee9e6935
For help, see: https://nodejs.org/en/docs/inspector
Welcome to Node.js v18.17.0.
Type ".help" for more information.
>

接著我們打開 Chrome 瀏覽器，並且在網址列輸入 chrome://inspect，接著你會看到以下畫面

畫面有一個 Remote Target，這個 Remote Target 就是我們剛剛在終端機輸入 node --inspect 所開啟的 Node.js，你可以點擊 inspect 來開啟 Chrome DevTools，……

……這個模式下用途是什麼呢？其實這個模式可以讓我們在 Node.js 中使用 Chrome DevTools 來偵錯，也就是說可以在這邊打上程式碼，並且透過 Chrome DevTools 來偵錯，這樣就不用再去使用終端機來偵錯了。(…略，參考 : Day2 - Node.js 基礎)

.6 node 中許多指令不常使用，在此介紹一個常用的方式，也就是透過 node 指令來執行 JavaScript 檔案，這樣就不用每次都要進入 REPL 模式，再輸入程式碼，而是可以直接執行 JavaScript 檔案。

例如: 請你先建立一個資料夾(方法有(1)&(2)， 參考下述Note)，並且在資料夾中建立一個 index.js 檔案，接著在 index.js 檔案中輸入以下程式碼

console.log('Hello World')

Note(便條、注意):

(1) 如果你熟悉指令模式的話，可以(在 C:\Users\…>＿) 輸入mkdir example來建立資料夾，接著輸入cd example進入資料夾，最後輸入touch index.js來建立index.js檔案。（需注意 Windows 沒有touch指令，可以使用echo "" > index.js來建立檔案） (2) 或者利用VS code來建立index.js檔案(和建立或開啟HTML檔案相同)。

.7 接著(在 C:\Users\…>＿) 先輸入 node ，然後拖曳 index.js 檔案到終端機中，這時候終端機會出現以下畫面

node /Users/xxxx/Desktop/example/index.js

接著按下 <enter>，你會看到終端機出現 Hello World ，代表已經成功執行 JavaScript 檔案！

.8 總結 .6-.7 : (1)在REPL 模式下(即進入node >＿)輸入console.log('Hello World') <enter>

或者 (2)在終端機模式下( C:\Users\…>＿)輸入node index.js<enter>，執行的結果相同，都是輸出 'Hello World'

3.1 基礎終端機指令

  • cd [路徑]：移動到指定路徑
    • ex: cd /Users/username/Desktop：移動到桌面
  • cd ..：回到上一層資料夾
  • mkdir [資料夾名稱]：建立資料夾
    • ex: mkdir my-project：建立 my-project 資料夾
  • touch [檔案名稱（副檔名）]：建立檔案
    • ex: touch index.js：建立 index.js 檔案
    • 需注意 Windows 系統不支援此指令，請使用 echo > [檔案名稱（副檔名）] 代替
  • ls：列出當前路徑下的所有檔案與資料夾
  • ll：列出當前路徑下的所有檔案與資料夾（包含詳細資訊）
